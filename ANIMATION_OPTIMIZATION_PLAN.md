# 性能优化计划 - 动画时长

## 当前问题

| 时长 | 数量 | 状态 | 建议 |
|------|------|------|------|
| 2000ms (2s) | 2 | ❌ 太长 | 改为 300ms |
| 1000ms | 5+ | ❌ 太长 | 改为 300ms |
| 700ms | 3+ | ⚠️ 有点长 | 改为 300ms |
| 500ms | 10+ | ⚠️ 边界 | 改为 300ms |
| 300ms | 20+ | ✅ 最佳 | 保持 |
| 200ms | 5+ | ✅ 最佳 | 保持 |

## 优化策略

### 第一优先级：立即改进
1. `duration-[2s]` → `duration-300` (图片缩放)
2. `duration-1000` → `duration-300` (进度条、过渡)
3. `duration-700` → `duration-300` (模态框、导航)

### 第二优先级：保持不变
- `duration-300` 保持（已是最佳）
- `duration-200` 保持（已是最佳）
- 某些特殊动画可以保留（如鸟群飞行 15s）

## 预期效果

- ✅ UI 响应更快
- ✅ 用户体验更流畅
- ✅ 符合 UI/UX 最佳实践
- ✅ 减少动画偏好用户的等待感

## 实施方式

使用 find & replace 批量修改：
- `duration-[2s]` → `duration-300`
- `duration-1000` → `duration-300`
- `duration-700` → `duration-300`
- `duration-500` → `duration-300` (大部分情况)

## 注意事项

- 保留特殊动画（如 15s 的鸟群飞行）
- 保留 CSS 关键帧动画的时长
- 只修改 Tailwind 的 duration 类
